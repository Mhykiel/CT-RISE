Form Information
How to Complete: Request Information

The 'Request Information' section collects basic details, including the form's version, the date it was filled out, and the name of the person submitting it. Please follow these instructions to complete each part:

  • Revision This is the current form version. Leave this as '1' unless you're updating a previous request.
  • Date Last Modified Choose the date you are filling out the form. This helps track when updates are made.
  • Your Full Name Enter your full name so we know who submitted the request.
  • Your Email Provide your email address for any follow-up communication.
Task Information
How to Complete: Task Information

The 'Task Information' section collects key details about the task you're requesting. Please provide clear and accurate information for each item to ensure the request is handled correctly.

  • System Name Enter the full name of the system related to this task, like 'FMS' for the Financial Management System.
  • Application Name Enter the name of the specific application, such as 'Payroll App' for the payroll part of the system.
  • Synopsis Provide a one-sentence summary of the task, like 'Update payroll data submission process.'
  • Description Give a detailed explanation of the task, including specific needs or expectations. This helps the team understand your request.
  • Criticality Select how critical this task is: non-critical, business critical, or mission critical. This helps prioritize it.
  • Recovery Instructions Describe any file recovery requirements, such as sending files one by one, in bulk, or merged.
Environments
How to Complete: Task Environments

In the 'Task Environments' section, select the environment tier where your task will apply. There are two main categories: Lower Tier and Upper Tier.

  • Lower Tier Environments Choose this if the task is for non-production workflows, like Development (DEV), Integration (INT), System Acceptance Testing (SAT), Load (LOD), or Training (TRN). These are used to build and test features or updates before moving to higher tiers.
  • Upper Tier Environments Choose this if the task involves environments near or in production, such as User Acceptance Testing (UAT), Pre-Production (TFP), or Production (PRD). These are stable environments for final testing or live use.
  • After selecting 'Lower Tier' or 'Upper Tier,' you'll see a list of specific environments for your choice.

Schedule
How to Complete: Task Schedule

In the 'Task Schedule' section, specify the days and times for the task to run. Make sure tasks are scheduled to finish before any downstream processes begin. Note: SeGov may adjust the schedule if over 50% of task runs result in no files being moved or may disable the task entirely if no files are moved during the full retention period (7, 30, 60, or 90 days). This retention period allows up to three attempts to identify issues.

  • Details Comment Add any special notes or requirements for the schedule in the 'Details Comment' box. Include exceptions or specific instructions here.
  • Week Day Select the days the task should run.
  • Time Slot Selection Choose start and end times for the task in 30-minute intervals, using a 12-hour AM/PM format.
  • Example For a task running Monday, Wednesday, and Friday from 9:00 AM to 5:00 PM, select those days and set the times accordingly.
Important
  • Schedule windows that will result in files being moved.
  • SeGov may adjust the schedule if over 50% of task runs result in no files being moved.
  • SeGOV may disable your task entirely if no files are moved during a task's full retention period (7, 30, 60, or 90 days).
  • Retention periods are selected to allow three attempts of a task run to identify issues.

Sources
How to Complete: Sources Section

In the 'Sources' section, provide details about each source involved in the task. Follow the instructions below to fill out each field in the table.

  • Details Comment Add any special instructions or requirements for each source here, like setup details or specific configurations.
  • Source Ordinal Enter a number to set the order of the sources. This is important if there are multiple sources.
  • Network Select 'On-Premise' if the source is within a local network, or 'Azure' if it's in the Azure cloud.
  • Direction Choose 'Push' to send files from the source or 'Pull' to retrieve files from it.
  • Source Host Enter the hostname or IP address for the source. This identifies the machine or server holding the files.
  • Relative Folder Path Specify the folder path relative to the source's root where the files are located.
  • Filemask Enter a filemask to specify which files to process, such as '*.txt' for text files or '*.csv' for CSV files.
  • Search Subfolders Check this box to include subfolders in the search, or leave it unchecked to search only the main folder.
  • Adding and Removing Sources Use 'Add Row' to add a new source or 'Remove' to delete an existing source row if it's not needed.

Trigger and End Files
How to Complete: Trigger and End Files

In the 'Trigger and End Files' section, you'll specify how `.END` files and triggers should be used. An `.END` file signals that a file or set of files is complete. For example, if files like `apple.txt`, `flour.txt`, and `caramel.txt` are processed together, they shouldn't move until an `.END` file, like `ingredients.END`, confirms the group is ready. Once this `.END` file is detected, SeGov converts it into a trigger file (e.g., `bake-pie-process.TRG`) to initiate the appropriate batch process. `.END` and trigger files can be empty or include metadata like required file lists, checksums, or other verification info.

  • Details Comment Add any extra instructions or details for the `.END` and trigger files, such as naming rules or special requirements.
  • Related File Group Enter the number for the related file group to link the trigger to the correct files.
  • Network Select where the files are stored: 'On-Premise' or 'Azure'.
  • Destination Host Enter the hostname or IP of the server where the `.END` or trigger file will be sent.
  • Relative Folder Path Provide the folder path where the `.END` file will be placed.
  • Original .END Filename Specify the name of the `.END` file that signals task completion.
  • Process Activated Name the process, script, or job triggered by the `.END` file.
  • Special Content Describe any special metadata or content in the trigger file.

Notification Emails
How to Complete: Notification Contacts

In the 'Notification Contacts' section, specify individuals or teams who will receive updates on the status of file transfers (success, failure, or both). Follow these steps to complete the section:

  • Details Comment Add any additional notes on the notification process, such as frequency or specific content preferences.
  • Related File Group Enter the ID or number of the file group for which this contact should receive notifications.
  • Notification Status Select when the contact should receive notifications: Failure, Success, or Both.
  • To Email Provide the recipient's email address, ensuring it is correct for notification delivery.
